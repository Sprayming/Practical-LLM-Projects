## 二、RAG 核心面试题

### 2.1 基础题

Q: RAG 全链路是怎么跑的？

参考回答（结合 Dental RAG）：
PDF → PyPDF2 解析文字 → 按段落/条款切分 → embedding 模型转向量 → ChromaDB 存储 → 用户问题 → embedding → 向量检索 top-k → 拼接 prompt → DeepSeek API 生成 → 返回答案 + 引用来源

你的项目对应：loader.py → chunker.py → embedder.py → vector_store.py → retriever.py → llm.py

Q: Chunking 策略怎么选？

通用文档：RecursiveCharacterTextSplitter
法律文书：按条款优先级切分（第X条 > 段落 > 句号）

关键话术：chunk_size=500 是一般起点，法律文书必须保留条款完整性，所以 separators 优先匹配第X条。

Q: Embedding 用的是什么？为什么不用 OpenAI？

用 sentence-transformers（shibing624/text2vec-base-chinese），中文语义理解优于通用模型，本地部署无需 API 调用，隐私安全。768 维，与 DeepSeek 兼容。

### 2.2 进阶题

Q: 如何控制 RAG 的准确性？

三个维度：
1. 检索质量：去重（按 content[:60] 去重）+ 相关性过滤（score < 0.2 丢弃）
2. 引用溯源：返回原始文档 ID + 页码 + chunk_index，用户可一键跳转原文
3. 幻觉控制：prompt 强制条文中没有规定时明确说未找到相关规定

## 三、记忆系统设计（高频考点）

### 3.1 分层记忆架构

用户提问
  ├── 短期记忆（最近3轮原文，约600 token）
  │     直接拼入 prompt，保证语境连贯
  ├── 实体画像（结构化 JSON，约100 token）
  │     精确事实召回，置信度合并防冲突
  ├── 中长期摘要（压缩文本，约200 token）
  │     LLM 自动压缩旧对话，保留法条引用
  └── 长期情景记忆（ChromaDB 向量库）
        按语义检索相关历史对话（top-3）

面试话术：短期解决语境连贯，画像解决精确事实，向量解决语义召回。三层各司其职。

### 3.2 冲突处理

每条事实带置信度：
- 用户明确说我怕狗 → confidence: 0.9
- 用户随口说狗很可怕 → confidence: 0.3（不覆盖旧信息）

合并规则：同 key 时高置信度覆盖低置信度。

### 3.3 遗忘机制（说得出就算加分）

90天未访问的情景记忆 → 自动压缩为一段摘要
180天未访问 → 标记为可归档
用户主动回忆某段旧记忆 → 重置其新鲜度

### 3.4 Token 预算分配

| 层级 | 预算 | 逻辑 |
|------|------|------|
| System Prompt | 100 | 固定 |
| 短期记忆（3轮） | 600 | 动态截断 |
| 实体画像 | 100 | 结构化 JSON |
| 情景记忆（检索） | 500 | top-3 |
| 检索文档 | 2000 | top-5 x 400，去重 |
| 用户输入 | 500 | 截断保护 |
| 总计 | 约 3800 | 预留回复空间 |

## 四、成本与性能面试题

Q: 一次 RAG 问答花多少钱？

以 DeepSeek 计费（输入 1元/百万 token，输出 2元/百万 token）：
输入 3800 token x 1元/百万 = 0.0038 元
输出 500 token x 2元/百万 = 0.001 元
单次成本约 0.005 元（不到半分钱）

关键话术：单次成本极低，但日活 1 万用户 x 10 轮/天 = 500 元/天。所以必须控制检索数量、去重、压缩历史。

Q: 性能瓶颈在哪？

LLM 生成：主链路，必须同步
向量检索：本地文件，无并发
实体提取：影子任务，可异步
摘要合并：条件触发

## 五、项目经验问答

### 5.1 Dental RAG（手写版）

为什么手写不用 LangChain？学习目的，每行代码都自己写过，出问题知道从哪查。
最大的坑？BOM 导致 .env 读不到、OpenAI SDK 版本兼容 DeepSeek API 的差异。
如何保证准确性？引用溯源、去重、score 过滤、prompt 防幻觉。
多用户怎么办？当前单用户，生产环境加 JWT + 多租户隔离。

### 5.2 Legal Document RAG（LangChain 版）

为什么用 LangChain？法律场景关注条款切分和引用精度。
记忆系统怎么实现的？三层 + 实体画像。
实体提取怎么做？每次对话后异步调 LLM 提取结构化事实，存 JSON 带置信度。
