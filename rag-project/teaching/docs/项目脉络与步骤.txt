═══════════════════════════════════════════════════════════
  项目一：企业 RAG 知识库问答系统 — 脉络与步骤指南
═══════════════════════════════════════════════════════════

项目位置：D:\rag-project
运行环境：Python 3.11+（你已装 3.13）
API 需求：DeepSeek API Key（仅LLM需要，嵌入本地运行）

┌─────────────────────────────────────────────────────────┐
│                   全流程数据流                           │
└─────────────────────────────────────────────────────────┘

  上传 PDF/DOCX 文档
         │
         ▼
  ┌─ loader.py ──────────────────────┐
  │  解析 PDF / DOCX / TXT / MD      │
  │  按页提取纯文本，保留页码信息      │
  └──────────────┬───────────────────┘
                 ▼
  ┌─ chunker.py ─────────────────────┐
  │  将长文本切分成 800 token 的小块  │
  │  块之间重叠 200 token             │
  │  避免"刚好切丢关键信息"           │
  └──────────────┬───────────────────┘
                 ▼
  ┌─ embedder.py ────────────────────┐
  │  调用 OpenAI Embedding API        │
  │  每段文本 → 1536 维向量           │
  └──────────────┬───────────────────┘
                 ▼
  ┌─ vector_store.py ────────────────┐
  │  存入 ChromaDB（本地向量数据库）    │
  │  持久化到 chroma_db/ 目录          │
  └──────────────────────────────────┘

  ────────── 到此"存进去"完成 ──────────

  用户提问："报销流程是什么"
         │
         ▼
  ┌─ embedder.py ────────────────────┐
  │  把问题也转成向量                  │
  └──────────────┬───────────────────┘
                 ▼
  ┌─ retriever.py ───────────────────┐
  │  去 ChromaDB 搜索最相似的 5 个块   │
  │  过滤掉分数低于 0.45 的结果        │
  │  返回 [(原文, 分数, 来源文档), ...] │
  └──────────────┬───────────────────┘
                 ▼
  ┌─ llm.py ─────────────────────────┐
  │  把找到的文本块 + 用户问题         │
  │  拼成 Prompt 发给 DeepSeek        │
  │  DeepSeek 生成带引用的回答         │
  └──────────────┬───────────────────┘
                 ▼
  返回给用户：回答 + 引用来源
