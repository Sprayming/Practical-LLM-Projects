二、RAG 核心数据流
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

摄取流程 (上传文档 → 入库):
  文件 → loader.py → LoadedDocument → chunker.py → TextChunk[]
  → embedder.py → 向量[] → vector_store.py → ChromaDB

  编排: pipeline.py 的 ingest_document() 串联以上4步

问答流程 (用户提问 → 回答):
  query → embedder.embed_query(query) → 向量
  → retriever.retrieve(query_vec) → 搜 ChromaDB
  → llm.generate(query, sources) → DeepSeek → 回答

3.7  ingestion/chunker.py — 文本分块器
────────────────────────

  class TextChunk: content, chunk_index, page_number, document_id

  def get_tokenizer(model):  ← tiktoken 编码器
  def count_tokens(text, enc):  ← token 计数

  def chunk_document(doc, chunk_size=800, chunk_overlap=200):
      for page in doc.pages:                 ← 逐页
          paragraphs = re.split(r"\n\s*\n", page.text)
          for para in paragraphs:            ← 逐段
              超长(>chunk_size) → 按句子切 + overlap
              正常 → 累积 buffer + 超限 flush + overlap
          flush 每页剩余 buffer
      return chunks
